# 《营造录：寒门赴考》技术实现文档

## 1. 技术选型

- **引擎**：Unity 2022 LTS（建议 2022.3.x）
- **语言**：C#
- **渲染管线**：Built-in 2D（首版），后续可迁移 URP 2D
- **目标平台**：Windows（比赛演示版）
- **版本控制**：Git + Git LFS（大图/音频可选）

## 2. 总体架构

项目按“核心层—玩法层—表现层—数据层”组织：

1. **Core（核心层）**
   - 游戏状态机（启动、主地图、关卡、结算）
   - 场景与存档（支持多分支数据）
   - **生存与经济状态机（名望、饱食度、劳累值、盘缠管理）**
2. **MapLogic（玩法层-主地图）**
   - 地块旋转（顺/逆时针）并暴露盲区（挂载不同面片的触发盒）
   - 邻接矩阵连通性校验
   - 关卡入口及隐藏关卡解锁条件
3. **MiniGames（玩法层-关卡）**
   - 民居：三视图拼装逻辑、滤水与平衡提水
   - 桥梁：轻塔防机关逻辑
   - 城关：双线联动机关逻辑
   - 皇宫连考：看图猜成语、路线沙盘迷宫、极速高难空间拼装
   - **隐藏关卡核心库：** 捕鱼QTE、三消矩阵、华容道（推箱子网格计算）、一笔画与刮刮乐
4. **UI（表现层）**
   - 对话系统（支持多条件分支、动态抛出Debuff警告）
   - 状态面板（盘缠、饱食度、劳累值、名望及相应特效光晕）
   - 图鉴系统（卡片展示、收集进度）
5. **Resources/Configs（数据层）**
   - 剧情文本、关卡参数、图鉴条目、经济数值表

## 4. 核心系统设计

### 4.1 大地图旋转连通系统（MVP 优先级 P0）

- 地图抽象为二维网格，每个格子保存：
  - 地块类型（道路/障碍/关卡入口）
  - 朝向（0/90/180/270）
  - 通路掩码（上右下左）
- 当玩家旋转地块时：
  1. 更新朝向与通路掩码
  2. 重建邻接关系
  3. 运行 BFS/DFS 判断起点到目标是否连通
- 连通成功后触发：
  - 路径高亮
  - 主角自动移动到下一章节入口

### 4.2 生存与经济数据同步（P1）
- 数据载体基于 `PlayerProgress` 序列化；
- 在 `GameManager` 提供统一下扣除与增加的接口方法，触发相应事件（`OnSatietyChanged`, `OnFatigueChanged`）。
- **拦截器机制：** 在玩家点击“进入重体力关卡”（如平衡提水）前，增加一个验证拦截网关，若饱食度 < 20% 强制弹出对话/提示框中止转场。

### 4.3 隐藏关卡与多路分支
- 大地图的3D地块模型每一侧可绑定不同的 Trigger Box；
- 当旋转至正对摄像机时（夹角判断或Raycast探测），激活相应的入口图标。

### 4.4 对话系统（JSON 驱动）

- 读取 `Resources/Configs/dialogue_*.json`
- 支持角色名、文本、立绘占位、分支节点（后续）
- UI 与逻辑解耦，便于后期替换美术

## 6. 开发里程碑（6 周）

- **W1-W2（P0）**：主地图旋转 + 连通性验证 + 基础角色移动
- **W3（P1）**：第一章民居关卡可通关版本
- **W4（P1）**：桥梁关卡 + 通用机关复用
- **W5（P2）**：城关/皇宫核心原型 + 图鉴系统
- **W6（P2）**：美术替换、打包、答辩演示流程固化

## 7. 质量保障

- 每个系统至少 1 个可复现场景（Demo Scene）
- 关键逻辑（连通、机关状态切换）添加日志与断言
- 提交规范：`feat/fix/chore + 模块名`

## 8. 风险与预案

- **风险**：地块旋转后碰撞与寻路不同步
  - **预案**：旋转后统一触发一次 `RebuildGraph()` 并延迟 1 帧刷新碰撞
- **风险**：玩法过多导致工期超载
  - **预案**：优先保证 P0/P1 完整闭环，P2 允许降级为展示原型
