# 从零开始开发路线（实操版）

## 第 0 步：准备环境（半天）

1. 安装 Unity Hub
2. 安装 Unity 2022.3 LTS（含 Windows Build Support）
3. 安装 Visual Studio 2022（勾选“使用 Unity 的游戏开发”）
4. 克隆本仓库并在 Unity Hub 中 `Add project`

## 第 1 步：先跑通最小闭环（2~3 天）

目标：只做一件事——**主地图旋转后，起点可到达终点**。

你需要完成：
- `MainMap` 场景
- 一个 5x5 地图测试网格
- 按键触发地块旋转（Q/E）
- 连通性日志输出（可达 / 不可达）

完成判定：
- 在 Play 模式下，旋转 1~3 次后能够出现“路径已连通”的日志

## 第 2 步：生存/盘缠系统与交互补齐（2~3 天）

- 给可旋转地块加高亮轮廓，并测试隐藏在盲区的入口触发逻辑。
- 搭建基础的全局玩家属性 UI：名望、盘缠、饱食度、劳累值，并完成数值的增减逻辑（包括 `GameManager` 数据存取和边界限制）。
- 连通成功后，主角自动走到下一节点并扣除体力和饱食度。

## 第 3 步：第一章民居关卡与小游戏群（1.5 周）

- 核心关卡：实现“三视图建筑拼图”。
- 分支玩法：实现简单的“倒水滤水”小游戏与“防抖第三人称提水”原型。
- 隐藏关卡：实现庭院背后的“捞鱼/钓鱼QTE”。

## 第 4 步：第二章与后续核心玩法（1.5 周）

- 第二章主线：塔防引擎搭建，敌人波次及阻挡/塌陷机关拦截。
- 第三章原型：处理森林冰火人一类的双线操作控制原型。
- 第四章原型：实现御花园看图猜词、沙盘走迷宫及最终的极速三视图拼装原型。

## 第 5 步：名望反馈与图鉴系统（1 周）

- 各种特殊状态判定：比如劳累值过高引发全局变暗发抖，或是名望高直接触发客栈免费剧情。
- 图鉴系统：每通关章节或完成隐藏关卡（如刮刮乐拓印、华容道）解锁卡片。
- 卡片信息、对话事件通过 JSON 读取，完成完整业务闭环。

## 日常开发节奏建议

- 每天结束前做一次可演示录屏（30~60 秒）
- 每个功能遵循：原型可用 -> 体验可玩 -> 视觉可看
- 优先把“完整流程”做通，再做细节优化

## 你接下来立刻要做的第一件事

打开 Unity，创建并保存场景：
- `Assets/Scenes/MainMap.unity`

然后告诉我你是否已经看到 Game 视图里运行的空场景。下一步我会继续手把手带你完成 5x5 地图与旋转逻辑。
