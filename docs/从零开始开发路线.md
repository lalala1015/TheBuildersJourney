# 从零开始开发路线（实操版）

## 第 0 步：准备环境（半天）

1. 安装 Unity Hub
2. 安装 Unity 2022.3 LTS（含 Windows Build Support）
3. 安装 Visual Studio 2022（勾选“使用 Unity 的游戏开发”）
4. 克隆本仓库并在 Unity Hub 中 `Add project`

## 第 1 步：先跑通最小闭环（2~3 天）

目标：只做一件事——**主地图旋转后，起点可到达终点**。

你需要完成：
- `MainMap` 场景
- 一个 5x5 地图测试网格
- 按键触发地块旋转（Q/E）
- 连通性日志输出（可达 / 不可达）

完成判定：
- 在 Play 模式下，旋转 1~3 次后能够出现“路径已连通”的日志

## 第 2 步：可视化与交互补齐（2~3 天）

- 给可旋转地块加高亮轮廓
- 连通成功后，主角自动走到下一节点
- 添加“鲁班令”消耗计数（UI 文本先占位）

## 第 3 步：第一章民居关卡（1 周）

- 先做纯逻辑拼图（无复杂动画）
- 构件拖拽到三视图槽位
- 全部正确 -> 判定通关 -> 返回主地图解锁新地块

## 第 4 步：第二章桥梁关卡（1 周）

- 搭建简单敌人波次
- 通用机关（阻挡/塌陷）接入
- 先保证“能玩”，再调手感

## 第 5 步：图鉴系统 + 包装（1 周）

- 每通关章节解锁 1 张建筑卡
- 卡片信息从 JSON 读取
- 主菜单中可查看收集率

## 日常开发节奏建议

- 每天结束前做一次可演示录屏（30~60 秒）
- 每个功能遵循：原型可用 -> 体验可玩 -> 视觉可看
- 优先把“完整流程”做通，再做细节优化

## 你接下来立刻要做的第一件事

打开 Unity，创建并保存场景：
- `Assets/Scenes/MainMap.unity`

然后告诉我你是否已经看到 Game 视图里运行的空场景。下一步我会继续手把手带你完成 5x5 地图与旋转逻辑。
